## Overview

Button is an essential element that provides feedback when clicked or tapped. It is a flexible interactive element
that fits into multiple contexts and UIs; such as **forms**, **cards**, **modals**, etc.

To apply Formare's standard button styling, just add `.Button` class to `<button>`, `<a>` or `<input>`.

<Preview lang="html">
  <PreviewDisplay>
    <div className="Buttons">
      <button className="Button">Basic Button</button>
      <button className="Button Button--primary">Filled Button</button>
      <button className="Button Button--primary Button--elevated">
        Elevated Button
      </button>
      <button className="Button Button--primary Button--outlined">
        Outlined Button
      </button>
      <button className="Button Button--primary Button--text">
        Text Button
      </button>
    </div>
  </PreviewDisplay>
  <PreviewCode>
    {`<button className="Button Button--primary">Themed</button>
      <button className="Button Button--primary Button--outlined">Outlined</button>
      <button className="Button Button--primary Button--text">Text</button>
    `}
  </PreviewCode>
</Preview>

We highly recommend you to append the `.Button--{theme}` class to "fill" the button with your preferred color theme.
Then, you can transform it into either **outlined**, **text** or **elevated** or **raised button** by appending the corresponding
class as needed. This way, users should be able to quickly recognize it as an interactive element.

## Themes

By default, append the `.Button--{theme}` alongside `.Button` to make your button appear filled with your preferred theme.

As explained above, we **highly recommend** you to append this class everytime you create a button to create a highly
distinguishable element that users will quickly recognize.

## Button tags

The `.Button` class is primarily used with `<button>` tag, but it also be used in conjunction with other tags such as
`<a>` and `<input>` depending on context.

Should you use `.Button` class with `<a>` tag, we highly recommend you to apply `role="button"` property to
properly assign their role as an interactive element rather than a link. You can see more
[here](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/button_role).

<!-- prettier-ignore -->
<Preview lang="html">
  <PreviewDisplay>
    <div className="Buttons">
      <button className="Button Button--primary">Default button</button>
      <a href="#" className="Button Button--primary" role="button">Link button</a>
      <input type="button" className="Button Button--primary" value="Input button" />
    </div>
  </PreviewDisplay>
  <PreviewCode>
    {`
    <button class="Button Button--primary">Default button</button>
    <a href="#" class="Button Button--primary" role="button">Link button</a>
    <input type="button" class="Button Button--primary" value="Input button" />
    `}
  </PreviewCode>
</Preview>

1. Themed button: `Button--{theme}`
1. Raised button: `Button--raised`

#### Raised

Append `Button--raised` property to make your button appear raised. If you prefer to have your button appear raised with shadow instead,
see **Elevated** section.

> `.Button--raised` modifier class can only be applied to themed buttons. A basic button cannot extend this modifier.

> By default, `<a>` tag doesn't support `disabled` attribute like `<button>` or `<input type="button" />` does. You will need to append
> `.Button--disabled` to achieve the desired effect of "disabled" state. `pointer-events: none`
