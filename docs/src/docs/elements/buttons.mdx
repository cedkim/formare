## Overview

Button is an essential element that provides feedback when clicked or tapped. Button is a flexible
interactive element that can fit into numerous contexts and UIs, usually used inside **forms**, **cards**,
**modals** among others.

To apply Formare's standard button styling, just add `.Button` class to either of these tags; `<button>`,
`<a>` or `<input>` depending on the context.

<!-- prettier-ignore -->
```html preview=true
<div className="Buttons">
  <button className="Button">Basic Button</button>
  <button className="Button Button--primary">Filled Button</button>
  <button className="Button Button--primary Button--elevated">Elevated Button</button>
  <button className="Button Button--primary Button--outlined">Outlined Button</button>
  <button className="Button Button--primary Button--text">Text Button</button>
</div>
```

> You might notice the example above uses a wrapper div using `.Buttons` class (notice the plural declaration).
> This is a helper class that you can use also to add spacing in between buttons.

We highly recommend you to append the `.Button--{theme}` class to "fill" the button with your
preferred color theme. Then, you can transform it into either **outlined**, **text** or **elevated**
or **raised button** by appending the corresponding class as needed.

## Themes

By default, append the `.Button--{theme}` alongside `.Button` to fill your button with the
preferred color theme.

As explained above, we **highly recommend** you to append this class everytime you create a button
to create a highly distinguishable element that users will quickly recognize as an interactive
element.

```html preview=true
<div className="Buttons">
  <button className="Button Button--primary">Primary</button>
  <button className="Button Button--secondary">Secondary</button>
  <button className="Button Button--warning">Warning</button>
  <button className="Button Button--danger">Danger</button>
  <button className="Button Button--light">Light</button>
  <button className="Button Button--lighter">Lighter</button>
  <button className="Button Button--dark">Dark</button>
  <button className="Button Button--darker">Darker</button>
</div>
```

## Button tags

The `.Button` class is primarily used with `<button>` tag, but it also be used in conjunction
with other tags such as `<a>` and `<input>` depending on context.

Should you use `.Button` class with `<a>` tag, we highly recommend you to apply `role="button"`
property to properly assign their role as an interactive element rather than a link. You can see
more [here](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/button_role).

```html preview=true
<div className="Buttons">
  <button className="Button Button--primary">Default button</button>
  <a href="#" className="Button Button--primary" role="button">
    Link button
  </a>
  <input
    type="button"
    className="Button Button--primary"
    value="Input button"
  />
</div>
```

## ⚠️ Experimental

Below is a list of experimental features of a button component. Use them at your own discretion!

#### Raised

Append `Button--raised` property to make your button appear raised. If you prefer to have your button
appear raised with shadow instead, see **Elevated** section.

> `.Button--raised` modifier class can only be applied to themed buttons. A basic button cannot
> extend this modifier.

```html preview=true
<button className="Button Button--primary Button--raised">Primary</button>
<button className="Button Button--secondary Button--raised">Secondary</button>
```
